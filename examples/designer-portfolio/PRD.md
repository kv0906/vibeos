# Portfolio Website — PRD

## Why

Designers without portfolios don't get hired. I need a home for my work that shows both *what* I design and *how* I think.

## What

A personal portfolio website with:
- Hero section (who I am, what I do)
- Selected work (3 case studies)
- About page (my story, process)
- Contact (simple form or email link)

## User Stories

- As a **recruiter**, I want to quickly see Linh's best work, so I can decide if she's worth interviewing
- As a **hiring manager**, I want to understand her design process, so I can assess her thinking
- As **Linh**, I want to easily update my portfolio, so it stays current

## Scope

**In:**
- Single-page or minimal pages (Home, Work, About, Contact)
- 3 case studies with process breakdown
- Mobile responsive
- Dark/light mode
- Simple animations (not flashy)

**Out:**
- Blog (v2)
- CMS (manual updates are fine for now)
- Analytics (can add later)

## Flow

```
Landing → Scroll to see work → Click case study → Read process → Contact
```

## Case Study Structure

Each case study follows:
```
1. Overview (problem, role, timeline)
2. Challenge (what was hard)
3. Process (research → ideate → design → test)
4. Solution (final designs with context)
5. Results (metrics or qualitative feedback)
6. Learnings (what I'd do differently)
```
